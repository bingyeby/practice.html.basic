Some patterns

  1. Except for the first line, lines set 'margin-right: -100%;' would automatically jump to the upper line.
  2. With the magic 'margin-right: -100%;' is all inline-block elements set this property would interact with each other as if they are on the same line!
  3. The red lines represent the the top and bottom edge of the line box(we should find which line box they represent). The distance between red lines is the height of the line box. Notice a inline elements' height of line box is implicit if you select the element in chrome dev tool, but you can apply 'display: inline-block' to the element and you would find the height of its line box.
  4. The green line with 'vertical-align: text-top;' represent the top edge of the content area of inline elements whose font-size is equal to its parent, likewise the green line with 'vertical-align: text-bottom;'.
  5. For inline elements, the text area your pointer could select is a part of the line box.
  6. For inline elements, the blue-covered area you see in Chrome Dev Tools if you select the element is the content area but, by definition, its height is mysterious 'auto', which is defined by its font-size and font-family, and excludes leading.
  7. For inline-block elements, the text area your pointer could select is the content area.
  8. For inline-block elements, the blue-covered area you see in Chrome Dev Tools if you select the element is the content box whose height is 'line-height', which means considering the leading, which could be negative. The 'vertical-align' property aligns the element using the margin box of its inline-block display (even the element is a inline element). ---from W3C
  9. For inline-block elements, the height of the text box of an inline-block element is greater than the height of the content area (including its line height) if the inline-block element is set 'line-height: 1;'
  10. The height of the line box is determined by its content. There is no property to set the line box's height.
    • If the line is empty its only content is the text-box. In this case the height of the line box equals the height of the text-box, i.e. the line-height. This is also true, if the line only contains text.
    • If the line has content, its content is vertically aligned in a way that it needs the least space. The resulting height is the height of the line box.
    • The line-height is the lower bound of the line box's height. The line box gets taller if its content would not fit into it otherwise.
